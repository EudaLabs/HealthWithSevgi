6. Key Decisions Made

  1. Clinical-first language throughout Steps 6–7. All feature labels use plain clinical names (e.g., “Serum Creatinine (mg/dL)”) instead of raw database column names. A 100+ entry CLINICAL_NAME_MAP translates every feature. Zero raw column names visible to the end user.
  2. Fast What-If analysis with predict_proba. Probability shifts are calculated using predict_proba instead of recomputing SHAP values, making What-If simulations near-instant (<100ms) compared to 3–5 seconds for full SHAP recomputation.
  3. Three-patient sampling strategy for waterfall explanations. Low-risk, mid-risk, and high-risk representative patients are sampled automatically from test data. This provides meaningful contrast in SHAP waterfall explanations without requiring the user to browse through hundreds of patients.
  4. Strict bias thresholding for fairness alerts. The bias auto-detection banner uses a >10pp sensitivity gap rule for triggering warnings, and a >15pp demographic gap for representation warnings. These thresholds were chosen to balance clinical significance against false alarms.
  5. LLM-powered clinical insights via InsightService. Integrated MedGemma and Gemini as LLM providers for generating domain-specific clinical interpretations of model results, enriching the EU AI Act checklist descriptions and case study analysis.
